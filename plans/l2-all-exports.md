# Добавление переменной __all__ (Задача L2)

## Summary
План закрытия отложенной минорной задачи **L2** из дизайн-ревью. В каждый из ~63 Python-модулей в `src/corpclaw_lite/` будет добавлена переменная `__all__`, чтобы явно изолировать публичный API (классы, функции, константы) от внутренних деталей реализации и импортов.

Подобный рефакторинг не меняет логику работы, но повышает читабельность кода и является хорошим тоном.

## Goals
- Просканировать все `.py` файлы (кроме тех 4, где `__all__` уже есть).
- Для каждого файла корректно определить список публичных сущностей (убрав приватные функции, классы-реализации и объекты `logger`).
- Добавить `__all__ = [...]` в начало каждого файла сразу после импортов.
- Убедиться, что `pyright strict` и `ruff` продолжают показывать 0 ошибок.

## Steps
1. Сохранить этот план в директорию проекта `plans/l2-all-exports.md`.
2. Подготовить локальный bash-скрипт / python-парсер (AST), который найдет все публичные объявления (`def func`, `class Class`, глобальные константы CAPS) в файлах и сгенерирует для них корректный блок `__all__`.
3. Точечно внедрить сгенерированные блоки в 63 файла или применить скрипт автоматической вставки. Скрипт будет обходить файлы, находить последний импорт и добавлять список.
4. Прогнать форматирование: `uv run ruff format src/` + `uv run ruff check src/ --fix`.
5. Прогнать проверку типов и тесты: `uv run pyright src/ && uv run pytest tests/ -v`.
6. Сверить отсутствие сломанных импортов и закоммитить изменения.
