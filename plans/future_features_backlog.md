# Бэклог будущих возможностей (на основе новых референсов)

## Состояние и цель
Документ содержит идеи и архитектурные решения, выявленные при анализе дополнительных референсных проектов (`nanobot`, `NanoClaw`, `ZeroClaw`). Эти функции могут быть добавлены в CorpClaw Lite на последующих этапах развития (Phase 6+) для расширения его возможностей в корпоративном сегменте.

---

## 1. Интеграция с корпоративными мессенджерами без Webhook (паттерн `nanobot`)
**Описание:** Реализация каналов (Slack, Feishu, DingTalk, WeCom) через WebSocket или long-polling соединения, а не через входящие HTTP-вебхуки.
**Бенефиты для CorpClaw Lite:**
- Отпадает необходимость выставлять входящие IP/порты агента наружу (крайне важно для закрытых корпоративных сетей и VPN).
- Агент сам инициирует исходящее соединение и может безопасно получать сообщения ("pull" модель).
**Статус:** 💡 Рекомендовано к добавлению в систему каналов (Phase 6).

## 2. Изоляция и инъекция секретов (паттерн OneCLI Agent Vault из `NanoClaw`)
**Описание:** Отказ от статического проброса `.env` переменных с чувствительными ключами (API, OAuth) при старте изолированных Docker-контейнеров субагентов.
**Бенефиты для CorpClaw Lite:**
- Секреты хранятся только в защищенном хранилище хостового сервера.
- Когда субагенту нужен токен, он запрашивает его через обязательный защищенный (HMAC) IPC-канал.
- Ключи существуют только в оперативной памяти процесса во время выполнения конкретного инструмента, радикально снижая вероятность их утечки при удачной атаке на контейнер.
**Статус:** 🛡️ Стратегическое улучшение безопасности (Phase 7).

## 3. Email-канал (IMAP/SMTP polling)
**Описание:** Подключение агента к выделенному корпоративному почтовому ящику (например, `assistant@company.local`).
**Бенефиты для CorpClaw Lite:**
- Позволяет агенту взаимодействовать с сотрудниками через привычную почту и автоматизировать сортировку, обработку входящих отчетов или документов от клиентов/подрядчиков по email.
**Статус:** 💡 Возможная интеграция в подсистему каналов связи.

## 4. Динамическая загрузка корпоративных стандартов (паттерн Qodo из `NanoClaw`)
**Описание:** Подгрузка внутренних стандартов написания кода, архитектурных правил или требований к документации перед выполнением задач программирования или ревью.
**Бенефиты для CorpClaw Lite:**
- Агент (или субагент) генерирует код и ревьюит пулл-реквесты, жестко руководствуясь специфичными правилами вашей компании, а не "усредненными" общими знаниями базовой LLM.
**Статус:** 🧠 Логичное расширение для скиллов (`code_reviewer.md`, `content_writer.md`).

## 5. Интерактивный Onboarding (паттерн `nanobot`)
**Описание:** Внедрение встроенного CLI-мастера начальной настройки (условно: `uv run corpclaw-lite onboard --wizard`).
**Бенефиты для CorpClaw Lite:**
- Диалоговый сбор параметров подключения к локальным LLM (Ollama/vLLM URL, выбор моделей, настройка роутинга) без необходимости вручную редактировать `settings.yaml` и создавать структуры `departments.yaml` с нуля.
**Статус:** 🛠️ Улучшение UX для администраторов.

## 6. Интеграция с периферийными устройствами (паттерн `ZeroClaw`)
**Описание:** Создание абстракции `PeripheralProtocol` по аналогии со стандартными каналами для взаимодействия с аппаратным обеспечением (IoT-датчики, системы контроля доступа).
**Бенефиты для CorpClaw Lite:**
- Открывает применение агента на складах, технологическом производстве или для управления физической инфраструктурой ЦОД (снятие показателей, перезагрузка оборудования).
**Статус:** 🔬 Экспериментальная функция для IoT сегмента.
