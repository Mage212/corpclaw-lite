# Roadmap

Дорожная карта CorpClaw Lite после релиза `0.1.4-beta`.

Этот файл фиксирует не только будущие планы, но и важные переносы: часть пунктов,
которые ранее были запланированы на `0.1.1`, по факту не вошла в релизы после
`v0.1.0` и перенесена в backlog.

---

## 0.1.4-beta — реализовано

Текущий beta-релиз сместил приоритет с планировщика задач на Web-канал, устойчивость
локальных LLM и инструменты для корпоративных рабочих процессов.

### Web-канал и рабочий интерфейс

- Production-ready Web-канал на `aiohttp`: login, HttpOnly cookie, CSRF, WebSocket,
  approvals, statusline и файловый API.
- React/Vite интерфейс в `frontend/web`: чат, markdown-ответы, файловый менеджер,
  preview, загрузка/скачивание, операции с файлами и persistent transcript.
- Русскоязычный premium UI: операционный центр, обзор выполнения, последние файлы и
  результаты, сворачиваемые панели, отдельное меню пользователя и явная новая сессия.
- Общая идентичность Web и Telegram через внутренний `users.id`: единый workspace,
  память и контейнер для одного человека.

### Local LLM performance

- `LLMRequestQueue` для ограничения реальной inference-конкурентности.
- `slot_affinity` для llama.cpp: sticky-слоты для активных пользователей и overflow-слот
  для вспомогательных вызовов.
- Экспериментальный persistent KV-cache через llama-server slot save/restore API.
- Trace/health события для очереди, слотов, cache restore/save и mismatch validation.
- Ручные live LLM тесты в `tests/live_llm/`, исключенные из обычного pytest-пула.

### Tools, subagents и research workflow

- `web_search` и усиленный `web_fetch(format=raw|text)` для host-side web research.
- Структурированный research workflow: `research_search`, `research_fetch_source`,
  `research_read_source`, `research_store_fact`, `research_list_facts`, `research_finalize`.
- Excel-инструменты `excel_inspect` и `excel_workbook`: инспекция структуры, чтение
  диапазонов, formula-aware режимы и заполнение шаблонов.
- Новый skill `excel_filler`; старые неиспользуемые skills `code_reviewer`,
  `content_writer`, `doc_writer` удалены.

### Security и надежность

- Усилены Web auth, RBAC, session handling, download grants и workspace boundary checks.
- Контейнерная сеть зафиксирована как pure deny-all: `network_mode: none`, без
  `config/network_policy.yaml` и allowlist-конфига.
- Telegram получил более устойчивый transport и защиту от утечек токенов в логах.
- Полная релизная проверка: `1017 passed, 1 skipped`, frontend build/test passed,
  pyright без ошибок.

## Отложено из старого плана 0.1.1

Ранее эти пункты были запланированы как `v0.1.1 — Планировщик и редакторы`, но фактически
не реализованы и не входят в `0.1.4-beta`. Их нужно рассматривать как backlog, а не как
обещание следующего релиза.

### SchedulerService

Единый фоновый asyncio-сервис для cron-задач и календарных напоминаний:

- `CronTasksWorker` для выполнения задач по расписанию через AgentLoop;
- `CalendarWorker` для детерминированных Telegram-напоминаний без LLM;
- хранение в `data/scheduler.db`;
- запуск в Telegram/Web runtime и корректный shutdown через `GracefulShutdown`.

Текущий статус: не реализовано.

### Cron-задачи

Пользовательский сценарий вроде «каждый понедельник в 9:00 отправляй отчёт»:
агент сохраняет расписание, а SchedulerService выполняет задачу.

Текущий статус: не реализовано.

### Календарь с напоминаниями

Создание событий и отправка напоминаний пользователю в Telegram/Web.

Текущий статус: не реализовано.

### Редакторы данных и расширенный HTTP

- `csv_editor` — редактирование CSV файлов: строки, столбцы, ячейки.
- `xlsx_editor` — точечное редактирование Excel файлов без пересоздания всей книги.
- `http_request` — расширенный HTTP-клиент для интеграций: POST, PUT, DELETE, headers, auth.

Текущий статус: не реализовано. Часть Excel-сценариев уже закрыта `excel_workbook`, но это
не полноценный универсальный `xlsx_editor`.

## Ближайшие кандидаты после 0.1.4-beta

Приоритеты ниже стоит выбирать по эксплуатационной ценности, а не по старой нумерации
`0.1.1`.

### 1. SchedulerService и напоминания

Вернуть старый план планировщика как отдельный релизный фокус, если пользователям нужен
автономный агент не только в ответ на сообщение, но и по расписанию.

Рекомендуемый минимум первой итерации:

- SQLite-модель scheduled tasks;
- deterministic reminders без LLM;
- ручные команды create/list/delete;
- безопасное ограничение департаментов и максимальной частоты запуска.

### 2. Web UI hardening

Продолжить развитие Web-канала после первого production-ready релиза:

- роли администратора и базовая админ-панель;
- управление web-пользователями из интерфейса;
- улучшение мониторинга активных запросов и контейнеров;
- экспорт/очистка истории web-сессий.

### 3. Data editing layer

Развить текущие `excel_workbook`, `table_query`, `convert_format` в более явную систему
редактирования данных:

- сценарии CSV/XLSX patch/edit;
- dry-run diff перед изменением;
- валидация формул и merged cells;
- унифицированный отчет «что изменено».

### 4. Research и knowledge workflows

Довести research-agent до более корпоративного режима:

- сохранение исследовательских dossiers;
- повторная проверка источников;
- режимы цитирования и confidence scoring;
- интеграция с корпоративными стандартами и внутренними базами знаний.

## Future ideas

Идеи ниже остаются потенциальными направлениями и не привязаны к конкретной версии.

### Новые скиллы

- `project_assistant` — управление задачами, чеклисты, таймлайны. Возможная интеграция
  с Jira/Confluence/GitLab через будущий `http_request`.
- `onboarding_buddy` — помощь новым сотрудникам: ответы по корпоративным правилам,
  навигация по документации.
- `legal_review` — юридический review документов: формулировки, риски, соответствие
  регуляторике.

### Новые инструменты

- `ocr_tool` — распознавание текста со сканов и фотографий документов.
- `archive_tool` — упаковка и распаковка ZIP/RAR архивов.

### Мультиканальность

CorpClaw Lite теперь поддерживает CLI, Telegram и Web. Следующий уровень — подключение
внешних корпоративных каналов без публичных входящих webhooks.

Архитектурное направление:

- pull-модель через WebSocket/long-polling, где агент сам инициирует исходящее соединение;
- новый базовый класс для push/pull каналов поверх существующего `Channel`;
- отдельная реализация каждого канала с минимальным boilerplate.

### Изоляция секретов

Секреты сейчас задаются через `.env` и доступны host-side компонентам. Для более строгой
корпоративной эксплуатации нужен Vault-подход:

- хранить секреты только в защищённом хранилище на хосте;
- выдавать временный token через HMAC-защищенный IPC;
- держать token только в RAM и только на время выполнения инструмента;
- очищать token после завершения операции.

### Email-канал

Подключение агента к выделенному корпоративному почтовому ящику:

- IMAP polling входящих писем;
- SMTP для ответов и уведомлений;
- обработка вложений PDF/Excel/Word;
- команды в теме или теле письма.

### Корпоративные стандарты

Файлы стандартов в `config/standards/`, которые подгружаются в релевантные скиллы и
субагенты:

- стиль кода компании;
- требования к API-документации;
- шаблоны договоров;
- правила именования в Git.

### Интерактивный onboarding-wizard

CLI-мастер для первичной настройки администратора:

- подключение к локальному LLM;
- проверка модели;
- настройка департаментов;
- создание первого пользователя;
- автогенерация `.env` и `settings.yaml`.
